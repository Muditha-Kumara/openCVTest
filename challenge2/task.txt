Description:
You are given a photo of a taped grid on a floor (see attached sample). Write a Python
program that:
1. Detects all grid intersection points (corners) in the image,
2. Finds the four outer corners of the grid,
3. Performs perspective rectification (homography) so the grid appears as a top-down,
rectangular view, and
4. Rotates/aligns the final image so that the grid’s horizontal and vertical lines align with
the image axes.

Handle noise/illumination (thresholding, morphology, normalization).

Minimum Requirements
1. Input: a single grid image (image1_1.png).
2. Corners: detect all internal intersections (coordinates in image space) and overlay

3. Outer corners: identify the four outer corners (TL, TR, BR, BL) and visualize them.
4. Perspective rectification: compute the homography and output a rectified top-down

5. Axis alignment: ensure the rectified image is aligned so grid lines are horizontal/vertical

Suggested Workflow (guidance)
1. Preprocessing
Optional downscale, grayscale, smoothing (GaussianBlur).
Contrast improvement (CLAHE) or illumination compensation.
Thresholding (global/adaptive) and/or Canny edges.
Morphology (open/close) to suppress floor texture noise.
2. Detect grid structure (two alternative strategies—use one or combine):
A. Lines → Intersections
HoughLinesP → split lines into near-vertical and near-horizontal
by angle.
Merge collinear fragments into consolidated lines
(clustering/averaging).
Compute intersections for each vertical × horizontal pair; keep
those within the image bounds.
B. Corners → Row/Column Grouping
Detect corners via goodFeaturesToTrack / Harris on a
thresholded/edge image.
Cluster points into rows and columns (e.g., DBSCAN/k-means on
1D x/y coordinates) → infer the grid intersections.
3. Find the outer four corners

Take extreme intersections (min sum → TL, max sum → BR, min (x−y) → TR,
max (x−y) → BL), or
Use convex hull + polygon approximation (approxPolyDP) to estimate the
grid boundary.
Order corners as TL, TR, BR, BL.
4. Perspective rectification
Choose target size: e.g., [W × H] = [num_cols * cell_size,
num_rows * cell_size].
Compute M = cv2.getPerspectiveTransform(src_pts, dst_pts)
and apply cv2.warpPerspective.
5. Axis alignment
Measure residual skew of horizontal lines (e.g., HoughLines again) and rotate

enforce alignment via your choice of destination points.
6. Results
Show intermediate outputs: edges, lines, corners, rectified,
